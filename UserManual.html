Welcome to the Declarative Footer Builder!

This guide will walk you through everything you need to know to create beautiful, responsive footers for your websites without writing a single line of code. Let's get started!

Getting Started: The Main Screen

When you open the builder, you'll see a few key areas:

  • The Main Canvas: This is the large grid area where you'll build your footer. It's like your digital playground!
  • Device Controls: At the top, you'll find buttons for Desktop, Tablet, and Mobile. This lets you see and design how your footer looks on different screen sizes.
  • Settings & Actions: Below the device controls, you'll find inputs to change the grid layout (rows, columns) and buttons for styling, saving, and loading your work.

1. Building Your Footer Layout

First, let's set up the basic structure of your footer.

Switching Between Devices

Click the Desktop, Tablet, or Mobile buttons to switch your view. Each device has its own independent layout, allowing you to create the perfect experience for every user.

Adjusting Rows, Columns, and Gaps

Use the input fields labeled Columns, Rows, and Gap to control the grid. For example, a "4x2" grid with an "8px" gap is a great starting point for a desktop footer.
Pro Tip: Start by designing your Desktop view first. Then, switch to Tablet and Mobile and click the Sync From Desktop button to copy your work over. This saves a lot of time!

2. Adding and Arranging Content

Once your grid is ready, it's time to fill it with content.

Adding a Content Box

Simply click on any empty, gray-dashed cell in the grid. A new content box (a "div") will appear, ready for you to customize.

Drag and Drop

To move a content box, just click, hold, and drag it to another empty cell. It's that easy!

Resizing a Box

Hover over a content box. You'll see small blue handles on the right, bottom, and corner. Click and drag these handles to make the box span multiple columns or rows.

3. Styling Your Content

This is where you bring your footer to life! You can style the entire footer background or individual content boxes.

Styling the Whole Footer

Click the Style Footer button. A modal will appear where you can change the background color, padding, and other overall styles for the entire footer area.

Styling a Single Content Box

Hover over any content box and click the paintbrush icon. This opens the powerful Style Editor.

Inside the Style Editor

The Style Editor has several tabs:

  • Content: Choose what kind of content to add.
    • Text: Write any text you want. You can also make it a clickable link.
    • Image: Add an image by pasting a URL or choosing from sample logos. You can also make the image a link.
    • Social Icons: Add a group of social media icons, complete with links.
  • Text Styling: Change font size, weight, color, and alignment.
  • Image Styling: Control how your image fits within its box (e.g., cover, contain), its size, and more.
  • Spacing: Adjust the padding (space inside the box) and margin (space outside the box).
  • Effects: Add borders, shadows, and rounded corners.

Styling Multiple Boxes at Once (Multi-Select)

Hold down the Ctrl key (or Cmd on Mac) and click on all the boxes you want to style together.
Once selected, click the new Style Selected Divs button that appears. Any style you apply will affect all the selected boxes!
To select a range of boxes quickly, click the first box, then hold Shift and click the last box.

4. Advanced Editing Features

The builder includes powerful features to make your workflow fast and efficient.

Undo and Redo

Made a mistake? No problem! Use the Undo and Redo buttons to step backward or forward through your changes.

Copy and Paste

Hover over a box and click the copy icon. All empty cells will turn gray, indicating they are ready for pasting.
Click on any gray empty cell to paste a perfect copy of your content and styles. The pasted box will have "-copy" added to its name and content to keep things unique.

5. Saving, Loading, and Downloading

Save your hard work and get the final code for your website.

Saving and Loading Your Work

Click the Save/Load button to open the Configuration Manager.
  • Save: Give your design a name and save it to your browser's local storage or download it as a JSON file to your computer.
  • Load: Load a previously saved design from your browser or upload a JSON file from your computer.
  • Presets: You can also load one of the pre-built sample footers to get a head start!

Previewing and Downloading Your Final Footer

The Live Footer Preview section at the bottom of the page shows you exactly what your footer will look like.
Use the Desktop, Tablet, and Mobile toggle buttons within the preview to check its responsiveness.
When you're happy with your design, click the Download File button. This will give you a single, ready-to-use HTML file with all the necessary HTML and CSS code embedded. You can use this file directly in your web project!

Technical Documentation

This section provides a technical overview of the Lightning Web Components (LWCs) used in the Footer Builder.

Component: divStyleModal

Purpose: A modal for styling individual divs within the footer grid.

Attributes:

  • currentDivId: The ID of the div currently being styled.
  • selectedDivIds: An array of IDs of the divs selected for bulk styling.
  • currentDevice: The current device view (e.g., 'desktop', 'tablet', 'mobile').

Methods:

  • connectedCallback(): Initializes the component by fetching the styles of the current div.
  • handleClose(): Closes the modal.
  • handleSave(): Saves the changes and closes the modal.
  • handleStyleChange(): Handles changes to the style inputs.
  • handleColorChange(): Handles changes to the color inputs.
  • handleSectionChange(): Handles switching between sections of the modal.

Events:

  • close: Fired when the modal is closed.

Component: footerConfigManager

Purpose: A modal for saving, loading, and managing footer configurations.

Attributes:

  • configurations: An object containing the footer configurations for all devices.

Methods:

  • handleCloseConfigModal(): Closes the modal.
  • handleConfigUpdate(): Updates the footer configurations.
  • handleSave(): Saves the current configuration to a file.
  • handleLoad(): Loads a configuration from a file.

Events:

  • close: Fired when the modal is closed.
  • configupdate: Fired when the configurations are updated.

Component: footerGeneratorParent

Purpose: The main component for the footer builder, containing the grid, controls, and modals.

Attributes:

  • columns: The number of columns in the grid.
  • rows: The number of rows in the grid.
  • gap: The gap between grid cells.
  • gridItems: An array of objects representing the cells in the grid.
  • previewMode: The current device view.

Methods:

  • connectedCallback(): Initializes the component.
  • switchDevice(): Switches the device view.
  • handleColumnsChange(): Handles changes to the number of columns.
  • handleRowsChange(): Handles changes to the number of rows.
  • handleGapChange(): Handles changes to the gap.
  • handleCellClick(): Handles clicks on grid cells.
  • handleDeleteCell(): Deletes a div from a cell.
  • openDivStyleModal(): Opens the divStyleModal.
  • openFooterStyleModal(): Opens the footerStyleModal.
  • openConfigModal(): Opens the footerConfigManager.
  • syncFromDesktop(): Syncs the content and dimensions from the desktop view.

Component: footerPreview

Purpose: Displays a live preview of the footer.

Attributes:

  • configurations: The footer configurations.
  • currentDevice: The current device view.

Methods:

  • refreshPreview(): Refreshes the preview.
  • generateCss(): Generates the CSS for the footer.
  • downloadFile(): Downloads the footer as an HTML file.

Component: footerState

Purpose: A service component for managing the state of the footer builder.

Methods:

  • setConfigurations(): Sets the footer configurations.
  • getConfigurations(): Gets the footer configurations.
  • updateGridItem(): Updates a grid item.
  • moveGridItem(): Moves a grid item.
  • updateFooterStyles(): Updates the footer styles.
  • setFormFactor(): Sets the form factor.
  • getFormFactor(): Gets the form factor.
  • subscribe(): Subscribes to state changes.
  • unsubscribe(): Unsubscribes from state changes.

Component: footerStyleModal

Purpose: A modal for styling the overall footer.

Attributes:

  • styles: The current footer styles.

Methods:

  • handleClose(): Closes the modal.
  • handleSave(): Saves the changes and closes the modal.
  • handleStyleChange(): Handles changes to the style inputs.
  • handleColorChange(): Handles changes to the color inputs.

Events:

  • close: Fired when the modal is closed.
  • styleupdate: Fired when the styles are updated.
